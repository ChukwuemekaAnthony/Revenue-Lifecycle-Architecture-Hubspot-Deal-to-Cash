System Design
Eight Data Domains

The architecture is organized into eight clearly bounded domains, each owning a set of objects with defined handoff triggers between them.

🏠
Customer Domain
Company · Contact
Root entities representing client organizations and stakeholder contacts. Parent to all downstream objects.
📈
Sales Domain
Deal
Commercial opportunity management. The deal_id originates here and propagates through the entire revenue chain.
📄
Contract Domain
Service Contract
Execution agreement created on Closed Won. Carries deal_id forward into all delivery and financial objects.
📍
Facility Domain
Facility
Physical locations belonging to client organizations. Assets are assigned to facilities for operational tracking.
⚙
Asset Lifecycle Domain
Asset
Individual equipment under management. Tracks lifecycle stage, utilization, service risk, and all date milestones.
🔧
Service Operations Domain
Maintenance Event
Single service or repair events on assets. Operational heartbeat of delivery. Triggers billing eligibility checks.
💴
Financial Domain
Invoice · Delivery Cost
Revenue and cost tracking both carrying deal_id. Enables gross margin calculation at deal and contract level.
🏭
Vendor Domain
Vendor · Vendor Service Contract
Equipment manufacturers and service providers. Managed through vendor service contracts tied to assets.

Deployment Plan
Seven Implementation Phases

Each phase depends on the prior phase. Blocker tasks are explicitly flagged. See implementation_checklist.md for full task detail.

01
CRM Foundation
  • License confirmation and portal setup
  • User roles and team configuration
  • Standard object property configuration
  • Duplicate detection rules
02
Custom Object Creation
  • Create all 6 custom objects
  • Configure all properties per catalog
  • Build all association rules
  • Create all pipeline definitions
03
Pipeline and Automation
  • Sales pipeline stage configuration
  • Deal Won to Contract workflow
  • Seven automation workflows built
  • Governance rule testing
04
Data Integration and API
  • Warehouse environment provisioned
  • Raw staging tables and API connections
  • Type casting and cleansing layer
  • Incremental load setup and monitoring
05
Warehouse Build
  • Dimension and fact tables created
  • SCD Type 2 logic implemented
  • Data quality checks passed
  • Revenue lineage validated
06
Analytics Enablement
  • BI platform connected to warehouse
  • Sales pipeline dashboard
  • Revenue and profitability dashboards
  • Operations dashboard and UAT
07
Governance and QA
  • Data ownership assigned per object
  • Quality monitoring scheduled
  • Access control configured
  • Final stakeholder sign-off
